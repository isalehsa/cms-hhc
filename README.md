# نظام إدارة الالتزام — نسخة Firebase

تطبيق ويب لإدارة الالتزام التنظيمي يعمل بالكامل على **الخطة المجانية (Spark) من Firebase** — بلا خادم إطلاقاً: الاستضافة على Firebase Hosting، والبيانات دائمة في Cloud Firestore، والمصادقة عبر Firebase Authentication، وكل المعالجة (قراءة PDF/Word، التعرف الضوئي OCR، التحليل الذكي، تصدير Excel) تجري داخل متصفح المستخدم.

يُضاف النظام أو اللائحة بنصّها الكامل — لصقاً أو تحميلاً كملف PDF أو Word (.docx) — فيقوم التطبيق آلياً بـ:

1. **استخراج جميع المواد والبنود** من نص النظام/اللائحة.
2. **فرز كل مادة**: تنطبق / لا تنطبق على المنشأة.
3. **تحديد درجة الخطر** (عالي / متوسط / منخفض) المترتبة على عدم الالتزام.
4. **تحديد الإدارة المالكة** المسؤولة عن تطبيق كل مادة.
5. **إتاحة تعديل كل ذلك يدوياً لمدير الالتزام** — الانطباق، الخطر، الإدارة، النص، المبرر — مع إمكانية إضافة مواد وحذفها وإعادة التحليل.

ومعه: **مكتبة الالتزام** (سجل موحد عبر كل الأنظمة مع بحث وتصفية وإحصائيات)، و**الربط ثنائي الاتجاه بين مواد الأنظمة** باقتراحات تشابه نصي محلية، و**تصدير Excel** بتنسيق يمين-لليسار مع تلوين درجات الخطر، و**OCR** للملفات الممسوحة ضوئياً (عربي + إنجليزي عبر tesseract.js داخل المتصفح).

## النشر خطوة بخطوة (مرة واحدة)

### 1) إنشاء مشروع Firebase

1. ادخل إلى [console.firebase.google.com](https://console.firebase.google.com) وأنشئ مشروعاً جديداً (الخطة المجانية Spark تكفي).
2. من **Build → Authentication → Get started** فعّل طريقة الدخول **Email/Password**.
3. من **Build → Firestore Database → Create database** أنشئ قاعدة البيانات (وضع Production — القواعد تُرفع من هذا المستودع).
4. من **Project settings (⚙) → Your apps → أيقونة الويب `</>`** سجّل تطبيق ويب جديداً وانسخ كائن `firebaseConfig`.

### 2) ضبط ملفات المشروع

- الصق قيم `firebaseConfig` في الملف **`public/js/firebase-config.js`** (هذه القيم عامة وغير سرية — الحماية الفعلية عبر قواعد أمان Firestore).
- ضع معرّف مشروعك في **`.firebaserc`** بدل `YOUR_FIREBASE_PROJECT_ID`.

### 3) إنشاء المستخدمين وأدوارهم

1. من **Authentication → Users → Add user** أنشئ حساب مدير الالتزام (بريد + كلمة مرور)، وحسابات الاستعراض إن أردت.
2. انسخ **UID** الخاص بكل مستخدم من نفس الشاشة.
3. من **Firestore Database → Start collection** أنشئ مجموعة باسم `users`، وأضف وثيقة لكل مستخدم:
   - **معرّف الوثيقة (Document ID)** = UID المستخدم.
   - الحقول:

     | الحقل | النوع | القيمة |
     |---|---|---|
     | `role` | string | `compliance_manager` لمدير الالتزام أو `viewer` للاستعراض |
     | `display_name` | string | الاسم الظاهر في الواجهة (مثل: مدير الالتزام) |

   أي مستخدم بلا وثيقة دور يُعامل تلقائياً كمستعرض (قراءة فقط).

### 4) النشر

```bash
npm install -g firebase-tools
firebase login
firebase deploy
```

سينشر الأمر الاستضافة **وقواعد أمان Firestore** معاً، ويعطيك الرابط العام بصيغة:
`https://YOUR_PROJECT_ID.web.app`

عند أي تعديل لاحق يكفي `firebase deploy` مرة أخرى — **البيانات في Firestore لا تتأثر بإعادة النشر**.

## التحليل الذكي (Claude API)

- يُدخل مدير الالتزام مفتاح Claude API من زر **⚙** أعلى الصفحة. المفتاح **يُحفظ في متصفحه فقط** (localStorage) ولا يُرسل لأي جهة غير Anthropic، ولا يراه بقية المستخدمين.
- **مع المفتاح**: تُحلَّل النصوص بنموذج Claude مباشرة من المتصفح (مخرجات مقيدة بمخطط JSON صارم) فيُستخرج كل بند مع تصنيفه ومبرره.
- **بدون مفتاح**: يعمل محلّل نصي احتياطي يقسّم النص إلى مواد (المادة/البند/الفصل...) بقيم افتراضية، وتُعلَّم جميع البنود «بحاجة لمراجعة» ليصنّفها مدير الالتزام يدوياً.
- التحليل يجري داخل صفحة المتصفح — لا تغلق الصفحة قبل اكتماله (تظهر حالة التقدم في الواجهة).

## الصلاحيات

| الدور | القيمة في وثيقة `users` | الصلاحية |
|---|---|---|
| مدير الالتزام | `compliance_manager` | إضافة وتعديل وحذف وربط وإعادة تحليل |
| مستعرض | `viewer` (أو بلا وثيقة) | استعراض وتصدير فقط |

القراءة تتطلب تسجيل الدخول — لا يوجد وصول عام للبيانات. القواعد كاملة في `firestore.rules`.

## ملاحظات وحدود

- **حصص الخطة المجانية لـ Firestore** (1 غيغابايت تخزين، 50 ألف قراءة و20 ألف كتابة يومياً) كافية جداً لهذا الاستخدام.
- **حد وثيقة Firestore هو 1 م.ب** — نص النظام الواحد يجب ألا يتجاوزه (لا يحدث عملياً؛ قسّم اللوائح الضخمة إن لزم).
- **OCR**: ملفات PDF بلا طبقة نصية تُعالج تلقائياً بالتعرف الضوئي داخل المتصفح. أول استخدام يُنزّل بيانات اللغة (~15 م.ب) في متصفح المستخدم، وتُخزَّن مؤقتاً بعدها. الحد الأقصى 40 صفحة لكل ملف، وننصح بمراجعة النص المستخرج لأن دقة OCR تعتمد على جودة المسح.
- **حجم الملفات**: حتى 25 م.ب للملف الواحد (PDF أو .docx — صيغة .doc القديمة غير مدعومة).
- نسخة الخادم السابقة (Express + Render) محفوظة في تاريخ Git إن احتجت الرجوع إليها.

## بنية المشروع

```
firebase.json            إعدادات النشر (الاستضافة + قواعد Firestore)
.firebaserc              معرّف مشروع Firebase
firestore.rules          قواعد أمان Firestore (الأدوار والصلاحيات)
public/                  التطبيق كاملاً (ملفات ثابتة)
  index.html
  styles.css
  js/
    firebase-config.js   ← إعدادات مشروعك (تُملأ يدوياً)
    app.js               الواجهة الرئيسية
    db.js                طبقة البيانات على Firestore
    auth.js              المصادقة (Firebase Auth)
    analyzer.js          التحليل الذكي (Claude API) + المحلل الاحتياطي
    extract.js           قراءة PDF/Word + OCR داخل المتصفح
    export.js            تصدير Excel داخل المتصفح
    similarity.js        اقتراحات الربط (تشابه نصي عربي محلي)
    meta.js              القوائم المرجعية (الإدارات، درجات الخطر، الانطباق)
```
