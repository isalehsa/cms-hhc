# CHCC — نظام إدارة الالتزام

تطبيق ويب لإدارة الالتزام التنظيمي: يُضاف النظام أو اللائحة بنصّها الكامل — لصقاً أو **تحميلاً كملف PDF أو Word (.docx)** — فيقوم النظام آلياً بـ:

1. **استخراج جميع المواد والبنود** من نص النظام/اللائحة.
2. **فرز كل مادة**: تنطبق / لا تنطبق على المنشأة.
3. **تحديد درجة الخطر** (عالي / متوسط / منخفض) المترتبة على عدم الالتزام.
4. **تحديد الإدارة المالكة** المسؤولة عن تطبيق كل مادة.
5. **إتاحة تعديل كل ذلك يدوياً لمدير الالتزام** — الانطباق، الخطر، الإدارة، النص، المبرر — مع إمكانية إضافة مواد وحذفها وإعادة التحليل.

## مكتبة الالتزام والربط بين الأنظمة

- **مكتبة الالتزام**: سجل موحّد يعرض جميع المواد والبنود عبر كل الأنظمة المسجلة، مع بحث وتصفية (النظام، الانطباق، درجة الخطر، الإدارة المالكة) وإحصائيات إجمالية.
- **الربط بين مواد الأنظمة**: يقترح النظام موادَّ مشابهة من بقية الأنظمة (تشابه نصي عربي محلي دون حاجة لاتصال خارجي)، ويستطيع مدير الالتزام ربط المادة بمواد الأنظمة الأخرى — الرابط ثنائي الاتجاه ويظهر في الجهتين وفي المكتبة والتصدير.
- **تصدير Excel**: تصدير سجل نظام واحد أو مكتبة الالتزام كاملة (ورقة موحدة + ورقة لكل نظام) بتنسيق يمين-لليسار مع تلوين درجات الخطر وأعمدة الروابط.
- **OCR للملفات الممسوحة ضوئياً**: ملفات PDF بلا طبقة نصية تُعالج تلقائياً بالتعرف الضوئي على الحروف (عربي + إنجليزي عبر tesseract.js). يتطلب أول استخدام اتصالاً بالإنترنت لتنزيل بيانات اللغة (تُحفظ بعدها في `data/ocr-lang`)، ويُنبَّه المستخدم لمراجعة النص المستخرج لأن دقة OCR تعتمد على جودة المسح.

## التشغيل

```bash
npm install
export ANTHROPIC_API_KEY=sk-ant-...   # اختياري لكنه موصى به (التحليل الذكي)
npm start
# ثم افتح http://localhost:3000
```

- **مع مفتاح API**: يُحلَّل النص بنموذج Claude (مخرجات منظمة وفق مخطط JSON صارم) فيُستخرج كل بند مع تصنيفه ومبرره.
- **بدون مفتاح API**: يعمل محلّل نصي احتياطي يقسّم النص إلى مواد (المادة/البند/الفصل...) بقيم افتراضية، وتُعلَّم جميع البنود "بحاجة لمراجعة" ليصنّفها مدير الالتزام يدوياً.

## الحسابات الافتراضية

| المستخدم | كلمة المرور | الصلاحية |
|---|---|---|
| `compliance` | `Chcc@2026` | مدير الالتزام — إضافة وتعديل وحذف |
| `viewer` | `Viewer@2026` | استعراض فقط |

تُغيَّر كلمات المرور عبر متغيري البيئة `CHCC_MANAGER_PASSWORD` و `CHCC_VIEWER_PASSWORD`.

## متغيرات البيئة

| المتغير | الوصف | الافتراضي |
|---|---|---|
| `ANTHROPIC_API_KEY` | مفتاح Claude API لتفعيل التحليل الذكي | غير مضبوط (يُستخدم المحلل الاحتياطي) |
| `CHCC_MODEL` | نموذج التحليل | `claude-opus-4-8` |
| `PORT` | منفذ الخادم | `3000` |
| `CHCC_DATA_DIR` | مجلد حفظ البيانات (ملف JSON) | `./data` |
| `CHCC_OCR_LANGS` | لغات OCR | `ara+eng` |
| `CHCC_OCR_MAX_PAGES` | أقصى عدد صفحات تُعالج بالـ OCR | `40` |

## واجهات البرمجة (API)

| الطريقة | المسار | الوصف | الصلاحية |
|---|---|---|---|
| POST | `/api/login` | تسجيل الدخول | عام |
| GET | `/api/regulations` | قائمة الأنظمة | عام |
| POST | `/api/regulations` | إضافة نظام وبدء تحليله | مدير الالتزام |
| POST | `/api/extract` | رفع ملف PDF/Word واستخراج نصه (multipart، حقل `file`، حتى 25 م.ب — يشمل OCR للملفات الممسوحة) | مدير الالتزام |
| GET | `/api/library` | مكتبة الالتزام: جميع المواد عبر كل الأنظمة | عام |
| GET | `/api/library/export.xlsx` | تصدير المكتبة كاملة إلى Excel | عام |
| GET | `/api/regulations/:id/export.xlsx` | تصدير سجل نظام واحد إلى Excel | عام |
| GET | `/api/regulations/:id/articles/:articleId/related` | اقتراح مواد مشابهة من بقية الأنظمة | عام |
| POST | `/api/regulations/:id/articles/:articleId/links` | ربط المادة بمادة في نظام آخر | مدير الالتزام |
| DELETE | `/api/regulations/:id/articles/:articleId/links/:targetArticleId` | فك الربط | مدير الالتزام |
| GET | `/api/regulations/:id` | تفاصيل النظام ومواده | عام |
| POST | `/api/regulations/:id/reanalyze` | إعادة التحليل | مدير الالتزام |
| PATCH | `/api/regulations/:id/articles/:articleId` | تعديل مادة (الانطباق/الخطر/الإدارة/النص/المبرر) | مدير الالتزام |
| POST | `/api/regulations/:id/articles` | إضافة مادة يدوياً | مدير الالتزام |
| DELETE | `/api/regulations/:id/articles/:articleId` | حذف مادة | مدير الالتزام |

قوائم الإدارات ودرجات الخطر قابلة للتخصيص في `src/meta.js`.

## ملاحظات على ملفات PDF و Word

- الصيغ المدعومة: **PDF** و **Word (.docx)**. صيغة `.doc` القديمة غير مدعومة — تُحفظ بصيغة .docx أولاً.
- ملفات PDF الممسوحة ضوئياً (صور بلا طبقة نصية) تُعالج تلقائياً بالـ OCR — أول استخدام يتطلب إنترنت لتنزيل بيانات اللغة، والدقة تعتمد على جودة المسح فيُعرض النص للمراجعة والتدقيق.
- بعض ملفات PDF العربية قد يخرج نصها بترتيب حروف مشوّه بحسب طريقة إنشاء الملف؛ لذلك يُعرض النص المستخرج في النموذج **لمراجعته وتصحيحه** قبل بدء التحليل.
